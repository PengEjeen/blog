# 8. 네트워크층: 패킷의 라우팅

## 8.1 라우팅 알고리즘

- 거리벡터 DV 최소-비용 트리 사용 - 벨만포드 알고리즘
- 링크-상태(LS) 라우팅: 최소비용트리와 포워딩 테이블을 작성하기 위해 - 다익스트라 알고리즘
- 경로 - 벡터(Path-Vector routing) 라우팅 - 최상의 스패닝 트리(최소비용트리가 아님). 방문할 최소 노드 수를 사용

## 8.3 유니캐스트 라우팅 프로토콜

- 거리 벡터 알고리즘기반 RIP
- 링크 상태 알고리즘을 기반 OSPF
- 자율 시스템 AS
    - 인터 도메인 - 안에서만
    - 인트라 도메인 -다른 도메인과
    - 각 ISP는 네트워크와 라우터를 제어할 때 자율 시스템
    - 각 AS에서는 ICANN에서 자율번호ASN 부여
- RIP
    - 소규모 네트워크에서만 사용
    - [AD(Adminisrator Distnace) / Metric(routing host)]
        - AD 1 = static
        - 110 = ospf
        - 120 = rip
    - AD값 큰거 작은거 있으면 작은 값으로 감 ospf,rip -> ospf
    - metric 1값이 가장 작다는 내용
- OSPF
    - 대규모 네트워크에 사용
    - packet 종류
        - hello packet
            - 이웃 라우터 찾고 그들 사이에 연결성 확보
            - 15초마다 물어봄, deadinterval 40이 default값
            - DR - 지정라우터, BDR 백업 지정 라우터, drother둘 다 아님
        - db description packet
            - 각 라우터들 간의 db 동기화 체크
        - Lsr packet 수신 보냄
            - 동기화 안되어있으면 ls 동기화 요청
        - Lsu packet 수신 받음
            - Lsr에 대한 응답
            - 변화감지 시 or 30분마다 보냄
        - Lsack packet 수신 받았다 표시
    - area 0 == router 알고리즘 적용 범위
    - DR BDR 선정

## 8.4 멀티캐스트 라우팅

### 8.4.1 유니캐스팅

- 하나의 발신지 네트워크와 목적지 네트워크(one to one)
- 전송자는 하나 받는 사람은 여러명?

### 8.4.2 멀티캐스팅

- 하나의 발신지와 목적지 그룹 -> 1대 다수
- 발신지 주소는 유니캐스트 주소이지만 목적지 주소는 하나 이상의 목적지 주소로 이루어진 그룹 형태

### 8.4.3 DVMRP

- 거리 벡터 멀티캐스트 라우팅 프로토콜(DVMRP, Distance Vector Multicast Routing Protocol) 은 RIP를 확장한 것
- 근원지 기반 트리 기법을 사용
- 세 가지 단계를 거쳐 발신지 기반 멀티캐스트 트리 생성
    - 역경로 포워딩 RPF -> 소스와 라우터 자체 사이의 최적의 소스기반 트리 생성
    - 역경로 브로드캐스팅 RPB -> 루트인 라우터와 인터넷 상의 모든 네트워크 간의 브로드캐스트 트리 생성
    - 역경로 멀티캐스팅 RPM ->역경로 멀티캐스팅 알고리즘 사용하여 멀티캐스트 트리 생성

### 8.4.4 MOSPF

- 멀티캐스트 개방 최단 경로 우선 OSPF 프로토콜의 확장 형태
- 발신지 기반 트리 기법

### 8.4.5 PIM

- 프로토콜 독립 멀티캐스트-> 유니캐스트 라우팅 프로토콜이 필요한 일반 프로토콜을 부르는 명칭
- 밀집모드와 성긴모드 DM,SM 사용할 수 있음.
- IPM(PIM - DM)

### 8.5 IGMP

- 그룹 멤버십 정보를 수집하기 위해 사용되는 프로토콜
- ICMP와 같이 IP의 보조 프로토콜

### 8.5.1 메시지

- 질의와 보고메시지와 같은 두 가지 형태의 메시지를 가짐
- 질의 메시지는 주기적으로 라우터에 의해 모든 호스트로 그룹 내 멤버에 대한 보고 받음. query, report

### 8.5.2 멤버십 정보의 전파

- 라우터가 트리의 자신의 레벨에서 호스트들과 다른 라우터로 부터 정보 수집. 이후 루트노드의 라우터에게 전파
- 루트노드의 라우터는 멀티캐스트 트리를 만들기 위해 멤버십 정보 모음

### 8.5.3 캡슐화

- IGMP 메시지는 프로토콜 필드 값은 2 TTL은 1로 설정된 IP 데이터그램으로 캡슐화
