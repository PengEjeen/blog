# 4. 스택

후입선출: 가장 최근에 들어온 데이터가 가장 먼저 나감.

## 4.1 연산

push(): 스택에 데이터 추가

pop(): 스택에서 데이터 출력 후 삭제

is_empty(): 스택이 공백상태인지 검사

is_full(): 스택이 포화상태인지 검사

create(): 스택 생성

peek(): 요소를 스택에서 삭제하지 않고 보기만 함

구현

- 1차원 배열 stack[]
- 스택에서 가장 최근에 입력되었던 자료를 가리키는 top 변수
- 가장 먼저 들어온 요소는 stack[0], 가장 최근에 들어온 요소는 stack[top]
- 스택 공백상태면 top = -1

동적 배열 스택

malloc()을 호출하여 실행시간에 메모리를 할당 받아서 스택 생성

## 4.2 괄호검사

조건

1. 왼쪽 오른쪽 괄호 개수 맞아야함
2. 왼쪽괄호가 오른쪽보다 먼저 있어야함
3. 괄호 사이에는 포함관계만 존재

개요

1. 문자열에 괄호를 차례대로 조사, 왼쪽괄호면 스택에 삽입, 오른쪽 괄호면 스택에서 top괄호를 삭제한 후 오른쪽 괄호와 짝이 맞는지 검사
2. 스택이 비어있으면 조건 1,2 위배. 괄호의 짝이 안맞으면 조건 3위배
3. 마지막 괄호까지 조사한 후 스택에 괄호 남아있으면 조건1에 위배

## 4.3 수식계산

컴퓨터에서의 수식 계산 순서

- 중위표기식 -> 후위표기식 -> 계산
- 모두 스택 사용

계산

- 왼쪽에서 오른쪽으로 스캔
- 피연산자면 스택에 저장, 연산자면 필요한 수만큼의 피연산자 스택에서 꺼냄
- 연산 실행 후 저장

중위표기식 -> 후위표기식

- 피연산자의 순서는 동일
- 연산자들의 순서만 다름

=> 연산자만 스택에 저장했다가 출력하면됨.

## 4.4 미로탐색문제

현재의 위치에서 가능한 방향을 스택에 저장해놓았다가 막다른 길을 만나면 스택에서 다음 탐색 위치를 꺼냄.
