# LangGraph

상태(state) 기반 노드 그래프로 체인보다 복잡한 LLM 워크플로우를 안정적으로 구성하는 라이브러리.

## 왜 Chain이 아닌 Graph?

- Chain은 선형 흐름(A → B → C)에 강함. 분기/조건/순환이 들어오면 코드가 복잡해진다.
- Graph는 **노드 + 엣지 + 공유 상태(state)** 모델로 분기·조건 라우팅·병렬 실행을 1급 객체로 다룬다.

## 자주 쓰는 패턴

1. **순차 체인** — 단순 변환 파이프라인
2. **조건 분기 (Router)** — LLM의 판단 결과로 다음 노드 결정
3. **map-reduce 병렬** — 대규모 작업을 분해 후 병렬 실행 → 결과 집계 (다음 글 참조)
4. **Human-in-the-loop** — 노드 사이에 사용자 confirm 삽입

## 기술 메모

- 상태(state)는 보통 **TypedDict** 로 명시
- 노드 함수의 반환값은 state의 부분집합 (자동 merge)
- 체크포인트(checkpointer)로 재실행/되감기 가능
