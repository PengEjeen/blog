# 2. 로컬에서 Keycloak 띄우기

Keycloak을 처음 테스트할 때는 `start-dev`만으로도 실행할 수 있다. 다만 실제 프로젝트에 붙일 목적이라면 처음부터 DB를 분리해서 보는 편이 낫다. Keycloak은 사용자, realm, client, role, session 같은 설정을 저장해야 하므로 운영에서는 외부 데이터베이스가 필요하다.

이 글에서는 PostgreSQL을 붙여 로컬 환경을 구성한다. 완전한 운영 설정은 아니지만, "컨테이너 하나 띄워서 로그인 화면만 보는 수준"보다는 실제 배포 구조에 가깝다.

## 2.1 docker-compose 구성
아래 구성은 Keycloak 서버와 PostgreSQL을 함께 실행한다. PostgreSQL은 Keycloak 설정과 사용자 데이터를 저장하고, Keycloak은 이 DB를 바라보도록 환경 변수를 지정한다.

```yaml
# docker-compose.yml
version: "3.8"

services:
  keycloak-db:
    image: postgres:16
    container_name: keycloak-db
    environment:
      POSTGRES_DB: keycloak
      POSTGRES_USER: keycloak
      POSTGRES_PASSWORD: keycloak123
    ports:
      - "5433:5432"
    volumes:
      - keycloak_db_data:/var/lib/postgresql/data

  keycloak:
    image: quay.io/keycloak/keycloak:26.0
    container_name: keycloak
    command: start-dev
    environment:
      KC_DB: postgres
      KC_DB_URL_HOST: keycloak-db
      KC_DB_URL_DATABASE: keycloak
      KC_DB_USERNAME: keycloak
      KC_DB_PASSWORD: keycloak123
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: admin123
    ports:
      - "8080:8080"
    depends_on:
      - keycloak-db

volumes:
  keycloak_db_data:
```

여기서 중요한 값은 `KC_DB`, `KC_DB_URL_HOST`, `KC_DB_URL_DATABASE`, `KC_DB_USERNAME`, `KC_DB_PASSWORD`다. Keycloak은 이 값을 기준으로 PostgreSQL에 연결한다. `KEYCLOAK_ADMIN`과 `KEYCLOAK_ADMIN_PASSWORD`는 최초 관리자 계정을 만들 때 사용된다.

`command: start-dev`는 개발 모드 실행 명령이다. 로컬 테스트에는 편하지만 운영에서는 그대로 쓰면 안 된다. 운영에서는 hostname, HTTPS, proxy header, health endpoint, metrics, cache 같은 설정을 분리해서 잡아야 한다.

## 2.2 실행

실행:
```bash
docker compose up -d
```

접속:
- Admin Console: `http://localhost:8080`
- 계정: `admin / admin123`

컨테이너가 정상적으로 올라오면 브라우저에서 Admin Console에 접속할 수 있다. 처음 접속하면 master realm의 관리자 콘솔이 보인다. `master` realm은 Keycloak 자체를 관리하기 위한 영역이므로, 애플리케이션용 설정은 별도의 realm을 만들어 관리하는 것이 좋다.

## 2.3 실행 확인
컨테이너 상태는 아래 명령으로 확인한다.

```bash
docker compose ps
```

로그는 다음처럼 확인한다.

```bash
docker compose logs -f keycloak
```

정상 실행되면 Keycloak이 HTTP 8080 포트에서 요청을 받기 시작한다. DB 연결이 틀리면 Keycloak 컨테이너가 반복 재시작되거나, 로그에 PostgreSQL 연결 실패 메시지가 나온다. 이 경우 먼저 `KC_DB_URL_HOST` 값이 compose service 이름인 `keycloak-db`와 일치하는지 확인한다.

## 2.4 로컬 설정에서 주의할 점
로컬에서는 `localhost:8080`으로 접근하지만, 컨테이너 내부 네트워크에서 Keycloak이 DB에 접근할 때는 `localhost`가 아니라 `keycloak-db`를 사용한다. 컨테이너 안에서 `localhost`는 자기 자신을 의미하기 때문이다.

또 하나 자주 놓치는 부분은 볼륨이다. `keycloak_db_data` 볼륨을 사용하면 컨테이너를 내렸다 올려도 realm과 client 설정이 유지된다. 반대로 완전히 초기화하고 싶다면 컨테이너만 지우는 것으로는 부족하고 볼륨까지 삭제해야 한다.

```bash
docker compose down -v
```

이 명령은 DB 데이터까지 제거하므로 테스트 환경에서만 사용한다.
