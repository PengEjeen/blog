# 2. 순환

알고리즘이나 함수가 수행 도중에 자기 자신을 다시 호출하여 문제 해결하는 기법

순환

- 순환적인 문제에서는 자연스러움
- 함수 호출의 오버헤드(overhead) - 어떤 처리를 하기위해 사용되는 간접적인 메모리와 처리 시간

반복

- 수행속도 빠름
- 순환적인 문제에 대해서 프로그램 작성 어려울 수도.

## 2.1 거듭제곱

반복과 순환 비교

순환 - 시간복잡도 O(logn)

반복 - 시간복잡도 O(n)

순환이 더 효율이 좋음

## 2.2 피보나치수열

순환 - 2^n

반복 - n

순환이 비효율적임

## 2.3 하노이탑

2^n
