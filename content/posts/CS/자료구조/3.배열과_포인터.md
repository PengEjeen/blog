# 3. 배열과 포인터

## 3.1 배열

같은 형의 변수를 여러 개 만드는 경우에 사용

구조체: 타입이 다른 데이터를 하나로 묶는 방법

배열: 타입이 같은 데이터들을 하나로 묶는 방법

### 3.1.1 다항식

1. 다항식의 모든 항을 배열에 저장

- 모든 차수에 대한 계수값을 배열로 저장
- 하나의 다항식을 하나의 배열로 표현

2. 다항식의 0이 아닌 항만을 배열에 저장

- 다항식에서 0이 아닌 항만을 배열에 저장
- 계수, 차수 형식으로 배열에 저장

### 3.1.2 희소행렬

배열을 이용하여 행렬을 표현하는 2가지 방법

1. 2차원 배열을 이용하여 배열의 전체 요소를 저장

- 장점: 행렬의 연산들을 간단하게 구현 가능
- 단점: 대부분의 항들이 0인 희소행렬의 경우 많은 메모리 공간 차지

2. 0이 아닌 요소들만 저장

- 장점: 희소행렬의 경우, 메모리 절약
- 단점: 각종 행렬 연산들의 구현 복잡

희소행렬: 대부분의 항들이 0인 배열

## 3.2 포인터

다른 변수의 주소를 가지고 있는 변수

&: 변수의 주소를 추출

*: 포인터가 가리키는 곳의 내용 추출

## 3.3 동적 메모리 할당

- 프로그램의 실행 도중에 메모리 할당
- 필요한 만큼만 할당, 필요한 때에 사용하고 반납
- 메모리를 매우 효율적으로 사용 가능
