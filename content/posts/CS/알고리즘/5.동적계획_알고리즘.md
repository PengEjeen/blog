# 1.동적계획법

정의

하위 구조에 있는 부분 문제의 답을 기반으로 전체 문제의 답을 구하는 방법

특징

- 분할정복 알고리즘은 부분 문제의 해를 중복사용하지 않지만 동적 계획 알고리즘은 중복사용함.
- 분할정복은 큰 부분에서 작은 부분, 동적계획법은 작은 부분부터 큰 문제로
- 메모제이션을 이용, 한번 계산된 값을 캐시공간에 저장

## 1.1모든 쌍 최단 경로

### 1.1.1플로이드 알고리즘

- 모든 쌍 최단 경로를 찾는 동적 계획 알고리즘
- 시간복잡도 O(n^3)
- 매우 간단

방식

전부 비교

## 1.2 벨만 포드

최단경로 알고리즘

- 가중치를 갖는 방향성 그래프에서 최단 경로 문제를 풀기 위한 알고리즘
- 단일 시작점으로 시작, 각 정점에 이르는 최단 경로 구함
- 도달 가능한 지점으로의 최단 경로의 길이 최대 = 정점 수-1
- 다익스트라보다 오래걸리나, 간선의 가중치가 음수 가능
- 음의 사이클 존재 여부

시간복잡도

O(|V||E|)

방식

1. 각 정점에 대해 거리 매우 큰 값으로 초기화
2. 정점 수만큼 반복하면서 모든 간선에 대해 최단거리 값 업데이트
3. 음의 가중치를 가지는 순환 경로 존재하는지 확인
- > 모든 간선에 대해 간선의 시작 정점의 거리와 간선의 가중치 합이 간선의 도착 정점의 거리보다 작으면 음의 사이클이 존재함을 의미 (최소인데 또 돌렸다고 최소가 나올리가…)

## 1.3 연속 행렬 곱셈

개요

- 연속된 행렬들의 곱셈에 필요한 원소간의 최소 곱셈 횟수를 찾는 문제
- A*B*C를 할때 어떤걸 먼저하냐에 따라 곱셈 횟수가 차이남
- 부분 문제들이 겹쳐져 있음

방식

1. 배열의 대각선 원소들 0으로 초기화
2. for - 루프 L은 1 ~ n-1까지의 변화 -1 (n-1, n-2 . . .)
3. 최소를 찾으며 올라갑시다

시간복잡도

O(n^3)

## 1.4 편집 거리 문제

개요

- 하나의 문자열을 수정하여 다른 문자열로 변환시키기 위해 필요한 최소의 편집 연산 횟수
- 삽입, 삭제, 대체 연산이 사용

시간복잡도

O(mn)

방식

1. s -> s, st -> s str -> s 요런식으로 반복문 들어감
2. E[i,j] = min(E[i,j-1]+1, E[i-1,j]+1, E[i-1, j-1] + alpha)

## 1.5배낭 문제

개요

- 물건의 최대 가치를 찾는 문제
- 무게의 합이 C를 초과하지 말아야 함, 각 물건은 1개 씩만
- 배낭 용량이 커지면 알고리즘 수행시간이 너무 길어짐
- K[i-1, w-wi], K[i-1,w]가 미리 계산되어있어야 K[i,w] 계산 가능

방식

- 2차원 배열 K의 0번 행과 열을 0으로 초기화
- 물건 1~n고려 -> 배낭 용량 1~C 증가
- 내부 조건 -> 담아보려는 물건 i의 무게가 배낭용량w보다 크면 최대가치는 K[i-1,w]
- 고려하는 물건i의 무게가 w보다 같거나 작으면 담을 수 있음. i를 추가로 담으면 배낭 무게가 증가하니 w초과?

시간복잡도

- 하나의 부분문제 O(1)
- 전체 O(nC) C는 배낭 용량

## 1.6 동전 거스름돈 문제

개요

- 거스름돈으로 최소 수의 동전만을 거슬러주기
- 항상 최적해
- 배낭과 마찬가지로 1원씩 증가시켜 문제를 해결
- 부분문제 -> C[j] = min(1<=j<=k){C[j-di]+1}, if j>=dj d=동전종류 j=액수 반복문

방식

- 배열 C의 각 원소 inf로 초기화
- C[0]=0으로 초기화
- for 루프에서 액수 j를 1원부터 1씩 증가
- for루프에서 가장 큰 액면의 동전부터 1원짜리까지 차례로 동전 고려, 그 중 가장 적은 동전 수를 C[j]로
    - 거르름돈 액수인 j원보다 크지 않은 동전에 대해서만

그리디 알고리즘은 최적의 해 보장 못함

시간복잡도

O(nk)

거스름돈 j가 1~n원까지 변하며, 각각의 j에 대해서 최악의 경우 모든 동전을 1번씩 고려하기 때문
