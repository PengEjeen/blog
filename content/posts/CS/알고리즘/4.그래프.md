# 4.그래프

현상이나 사물을 정점과 간선으로 표현하는 것

그래프는 G로 표현하고 일반적으로 다음과 같이 표시

G = (V,E) == (정점들의 집합, 간선들의 집합)

무방향그래프

방향 그래프

가중치 그래프

네트워크

부분그래프 - 주어진 그래프의 정점과 간선의 집합으로 이루어진 그래프.

완전그래프 - 그래프에 속해 있는 모든 정점이 서로 연결되어있는 그래프

밀집 그래프 - 간선의 수가 최대 간선의 수에 가까운 그래프

희소 그래프 - 간선이 얼마 없음.

장단점

행렬 표현에 비해 공간의 낭비가 없다.

간선의 수가 적을 때 유용

거의 모든 정점 쌍에 대해 간선이 존재하는 경우 오버헤드가 크게 증가

인접행렬 - n개의 정점을 가진 그래프가 각각의 간선이 어디있는지 행렬로 표현

- 두 노드간의 간선을 직접 조회할 수 있다.

인접 리스트 - n개의 정점에 대해서 각각 하나의 연결리스트에 연결정보 담아서 구현

- 두 노드간의 간선을 직접 조회할 수 없고 순차 탐색을 해야한다.

간선 리스트 - 간선을 중심으로 표현.

## 4.1깊이우선 탐색(DFS)

- 루트를 시작으로 탐색하는 방법
- 루트의 자식정점을 하나 방문한 다음 최고 아래로감
- 내려갈 곳이 없으면 위로 되돌아오다가 내려갈 곳이 있으면 다시 내려가며 탐색

## 4.2 너비 우선 탐색 (BFS)

- 루트를 시작으로
- 루트의 자식을 차례대로 방문하며 탐색
- 루트에서 두 개의 간선을 거쳐서 도달할 수 있는 정점을 방문
- 루트에서 거리순으로 반복하여 방문하는 탐색방법

## 4.3최단경로찾기

가중치그래프의 출발점에서, 또 다른 도착점까지의 최단경로

**다익스트라 알고리즘, 벨만 포드 알고리즘**

### 4.3.1 다익스트라 알고리즘

한 정점에서 모든 다른 정점까지의 최단 경로

- * 최소신장트리와의 차이점
1. 프림 알고리즘은 **임의의 점에서 시작**, but 다익스트라는 **주어진 출발점에서 시작**
2. 프림 알고리즘은 **선분을 추가시킬 때**, 현재 상태의 트리에서 **가장 가까운 점을 추가**

but 다익스트라는 **출발점으로 부터 최단 거리가 확정되지 않은 점들 중 가장 가까운 점** 추가, **그 점의 최단 거리를 확정**

시간복잡도

O(n^2)

하지만, 프림 알고리즘과 같이 최소 힙을 사용하면 O(nlog(n))

## 4.4 부분 배낭 문제

짐을 쪼갤 수 없는 경우

0-1 배낭문제, 동적 계획법, 부분

짐을 쪼갤 수 있는 경우

부분배낭 문제, 그리디 알고리즘

### 4.4.1 부분 배낭 문제

- 단위 무게당 가장 값나가는 물건 먼저
- 안들어가는 경우 담을 수 있는 만큼만 물건을 부분적으로 담음.
1. 각 물건의 단위 무게 당 가치를 계산하여 정렬
2. while루프 - 단위 무게 당 값나가는 물건을 가져다 배낭에, 용량 초가되면 루프 종료
3. 현재 무게 w가 배낭의 용량 C보다 작으면, 해당 물건을 C-w 만큼만 담고, C-w만큼의 x의 가치 증가

시간복잡도

O(nlog(n))

응용

조합론, 계산이론, 암호학, 응용수학

### 4.4.2 집합 커버 문제

- 어떠한 전체 집합과 그 집합의 부분집합들이 주어졌을 때,

부분 집합들 중에서 가능한 한 적은 집합을 골라서 그 집합들의 합집합이 원래 전체 집합이 되도록 집합 선택하는 문제;

- n이 커지면 최적해를 찾는 것은 실질적으로 불가능

이를 극복하기 위해, 최적해가 아닌 근사해를 찾는 것…

시간복잡도

O(n^3)

### 4.4.3 작업 스케줄링

- 작업의 수행 시간이 중복되지 않도록 모든 작업을 가장 적은 수의 기계에 배정하는 문제
- 작업의 수와 각 작업의 시작시간과 종료시간으로 구성
- 시작 시간, 종료 시간, 작업 길이
    - 빠른 시작 시간 작업 우선 - 최적의 해 가능 나머지는 불가능
    - 빠른 종료 시간 작업 우선
    - 짧은 작업 우선 배정
    - 긴 작업 우선 배정

순서

1. 시작시간에 대해 작업 오름차순 정렬
2. while (작업이 다 배정될 때까지)
3. 가장 이른 시작시간을 가진 작업 tj를 선택
4. 작업 tj를 수행 시간이 중복되지 않게 수행할 기계를 찾고, 기계가 있으면 배정
5. 기존의 기계들에 배정할 수 없는 경우에는 새로운 기계에 배정
6. tj 배정 후 배열에서 제거
7. 각 기계에 배정된 작업 순서를 리턴

시간복잡도

- n개의 작업을 정렬하는데 O(nlog(n)) 시간 소요
- while에서 O(m)
- while n번 시행되므로 O(mn)
- so O(nlog(n)) + O(mn)

### 4.4.4 허프만 압축

- 파일의 각 문자는 일반적으로 고정된 크기의 코드로 표현 -> 아스키코드8bit * 파일의 문자 수
- 효율적인 메모리공간을 위해 단축
- 무손실 압축에 쓰이는 엔트로피 부호화의 일종
- 문자의 등장 빈도에 따라 다른 길이의 부호 사용
- 허프만으로 변환시킨 문자 코드들 사이에는 **접두부 특성**이 존재
- 접두부 특성의 장점 == 코드와 코드 사이를 구분할 특별한 코드가 필요 없음.

방식

1. 각 문자 당 노드, 그 문자의 빈도수를 노드에 저장
2. n개의 노드의 빈도수에 대해 우선순위 Queue 만든다.
3. while(Q에 있는 노드 수>=2)
4. 빈도수가 가장 작은 2개의 노드 Q에서 제거
5. 새 노드 N, 생성, A와 B를 N의 자식 노드로
6. N의 빈도수 = A와 B의 빈도수
7. 노드 N을 Q에 삽입

시간복잡도 nlog(n)
