## 3.2최소 신장 트리

주어진 가중치 그래프에서 사이클이 없이 모든 점들을 연결시킨 트리들 중 선분들의 가중치 합이 최소인 트리

**사이클이 없도록 모든 점을 연결**

- **‘최소’는 비용적인 측면을 의미**

종류

- 크러스컬
- 모든 비용을 순차적으로 나열하여 가장 적은 비용이 드는 간선들을 선택
- 프림
- 임의의 꼭지점(정점)을 선택, 연결된 가장 적은 비용의 정점을 선택

### 3.2.2 크러스컬 알고리즘(Kruskal Algorithm)

- 최소 신장 트리를 형성하기 위한 알고리즘
- 간선을 하나씩 선택
- 모든 간선을 가중치 순으로 정렬
- if(사이클 안생기면) 선분 추가
- 모든 정점 선택할 때까지 (1)을 반복

시간복잡도 O(mlogm)

- 선분들을 가중치로 정렬하는데 O(mlogm) 소모
- T 초기화하는 것이므로 O(1)시간 소모

### 3.2.3프림 알고리즘(Prim Algorithm)

- 가중치가 있는 연결된 무방향 그래프의 모든 꼭지점을 포함하면서 각 변의 비용의 합이 최소가 되는 부분 그래프인 트리

방식

1. 임의의점 하나 선택, n-1개의 선분을 하나씩 추가시켜 트리를 생성
2. 새로운 선분을 추가하면서 연결시킬 때 항상 최소의 가중치로 연결되는 선분을 추가
3. 연결되지 않은 정점들은 큐에 배정되어있다.
4. 각 정점들은 key값을 가지고, 인접한 정점 중 최소 비용으로 이동 가능한 정점 선택

시간복잡도

1. O(n^2)

Vmin을 찾는데 O(n) 시간소모

따라서 프림 알고리즘의 시간 복잡도: (n-1)*O(n) = O(n^2)

1. O(nlog(n))

배열 D를 힙 자료구조로 표현하고 최소 힙을 사용하여 Vmin을 찾으면 mlog(n) 가능

### 3.2.4크러스컬과 프림 알고리즘의 수행 과정 비교

- 크러스컬 알고리즘에서는 선분이 1개씩 추가, so n개의 점들이 각각의 트리인 상태에서 선분이 추가되면 2개의 트리가 1개의 트리로 합쳐지는 것과 같다.
- 크러스컬은 n개의 트리들이 점차 합쳐져서 1개의 신장트리 만들어짐.
- 프림은 1개의 트리가 자라나서 신장트리 만들어짐
