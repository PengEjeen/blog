# 2.정렬 알고리즘

- 선택, 버블, 삽입, 쉘, 힙, 합병, 퀵

## 2.1 선택정렬

개요

- 해당 위치에 맞는 자료를 선택하여 위치를 교환하는 정렬
- 제자리 정렬 알고리즘
- 작은 파일들에서 잘 동작

방식

1. 전체에서 최솟값 선택
2. 0번 원소와 자리 바꿈, 다음 최솟값은 1번 원소와 바꿈
3. 반복

장점

- 원리가 간단하여 구현이 쉬움
- 부가적인 저장공간이 요구되지 않음

단점

- 확장성이 좋지 않다

시간복잡도

O(n^2)

- 항상 일정한 시간 복잡도 -> 입력에 민감하지 않은 알고리즘

## 2.2 버블 정렬

개요

- 두 이웃하는 숫자를 비교하여 작은 수를 앞쪽으로 이동
- 속도가 느린 편, 코드가 단순하여 자주 사용됨.
- 비교 데이터가 많아지면, 성능 저하됨

방식

- 1,2 2,3 숫자 비교 -> 배열의 수만큼 반복

시간복잡도

- O(n^2)

## 2.3 삽입 정렬

개요

- 배열을 정렬된 부분과 정렬되지 않은 부분으로 나누고, 정렬되지 않은 부분의 가장 왼쪽 원소를 정렬된 부분의 적절한 위치에 삽입하여 정렬

방식

- 정렬 안 된 부분의 숫자 정렬된 부분에 삽입
- 반복하면 정렬 안 된 부분에는 아무것도 없음
- 정렬은 배열의 첫 번째 원소만이 정렬된 부분임.

시간복잡도

- O(n^2)

## 2.3 쉘 정렬

개요

- 삽입 정렬의 문제점을 개선한 정렬 -> 삽입할 원소의 자리가 멀리 떨어진 경우 한 칸씩 이동하는 방식 때문에 불필요하게 많은 반복과 비교가 발생
- 삽입 정렬이 어느 정도 정렬되어 있는 배열에 대해서는 대단히 빠르다는 점에 착안
- 삽입 정렬에 분할 접근 방식의 접근법 적용

방식

- 일정한 간격으로 떨어져 있는 자료들 끼리 부분 집합 구성, 부분 집합에 있는 원소들에 대해서 삽입 정렬 수행
    - 전체 리스트를 일정 간격의 부분 리스트로
    - 간격 줄임
    - 간격이 1이 될 때까지 부분 리스트의 삽입정렬 반복
- 뒷부분의 작은 숫자를 앞부분으로 빠르게 이동, 동시에 앞부분의 큰 숫자는 뒷부분으로 이동
- 일정 간격 떨어진 원소들끼리 먼저 정렬, 이후 전체 정렬

부분집합 선택

- n보다 작은 크기의 값을 간격으로 설정 -> 단계 거칠 때 마다 1/2만큼 감소
- Gap의 크기가 1일 때인 마지막 단계에서 쉘 정렬은 삽입 정렬 수행

시간 복잡도

- 수행 속도는 간격 선정에 따라 좌우된다.
- 쉘 정렬의 최악 경우의 시간복잡도 O(n^2)
    - 히바드 간격인 2k-1 사용하면 O(n^1.5)됨
- 이제 O(n1.25)으로 알려져 있다.

## 2.4 힙 정렬

개요

- 고급 정렬 알고리즘 중의 하나
- 이진 힙 드리 구조 사용
- 제자리 알고리즘

방식

- 오름차순의 정렬을 위해 입력 배열을 먼저 큰 숫자가 높은 우선순위를 갖는 최대 힙 구성
- root의 숫자를 힙의 가장 마지막 노드에 있는 숫자와 교환
- root에 새로 저장된 숫자로 인해 위배된 힙 조건을 해결하여 힙 조건 만족시키고, 힙 크기를 1개 줄임
- 이 과정을 반복하여 나머지 숫자 정렬
1. 주어진 배열을 최대힙으로 만든다
2. Delete와 Downheap을 통해 정렬
3. 맨 위와 맨 아래 바꾸고 맨 위 빼고 정렬 반복

시간복잡도

- 힙만드는데 O(n)
- 변수 초기화하는데 O(1)
- DownHeap .O(log n)
- O(nlogn)
