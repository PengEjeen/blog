# 1. Abstract
다중 에이전트 LLM 프레임워크는 대규모 언어 모델(LLM) 기반 에이전트 시스템 개발을 가속화하는 데 널리 사용됩니다. 이러한 프레임워크는 에이전트 간 상호 작용, 정보 저장 및 작업 조정 방식을 규정하는 고유한 아키텍처 구조를 갖습니다. 
그러나 시스템 성능에 미치는 영향은 아직 제대로 이해되지 않고 있습니다. 아키텍처 선택만으로도 지연 시간과 처리량에 상당한 차이가 발생할 뿐만 아니라 정확도와 확장성에도 큰 영향을 미칠 수 있기 때문에 이러한 이해 부족은 매우 중요합니다. 
이 문제를 해결하기 위해서는 
(i) 오케스트레이션 오버헤드, 메모리 동작, 계획, 특수화 및 조정과 같은 여러 기능을 종합적으로 평가하고, 
(ii) 아키텍처 효과를 분리하기 위해 제어된 프레임워크 수준의 조건에서 이러한 평가를 수행해야 합니다. 기존 벤치마크는 개별 기능에 초점을 맞추고 있으며 표준화된 프레임워크 수준 평가가 부족합니다. 
본 연구에서는 
(i) 기본 차원을 따라 다중 에이전트 LLM 프레임워크를 체계적으로 비교하기 위한 아키텍처 분류 체계를 도입하고, 
(ii) 기존 벤치마크를 표준화된 실행 파이프라인에 통합하는 통합 평가 도구인 MAFBench를 개발하여 이러한 한계를 해결합니다. MAFBench를 사용하여 여러 널리 사용되는 프레임워크에 걸쳐 통제된 실증 연구를 수행했습니다. 
연구 결과에 따르면 프레임워크 수준의 설계 선택만으로도 지연 시간이 100배 이상 증가하고, 계획 정확도가 최대 30% 감소하며, 조정 성공률이 90% 이상에서 30% 미만으로 떨어질 수 있습니다. 마지막으로, 이러한 연구 결과를 구체적인 아키텍처 설계 원칙 및 프레임워크 선택 기준으로 제시합니다.

# 2. Introduction
멀티 에이전트 LLM 프레임워크에서 instruction은
각 에이전트의 행동을 정의하고, 에이전트 간 상호작용을 조정하는 핵심 메커니즘이다.
instruction은 단순히 태스크 설명을 넘어서,
에이전트의 역할, 목표, 제약 조건, 그리고 다른 에이전트와의 협업 방식을 포함한다.
이러한 instruction 설계는 전체 시스템의 성능과 안정성에 직접적인 영향을 미친다.

## 1. Role Specification (역할 정의)

각 에이전트는 특정 역할을 부여받으며,
이 역할은 해당 에이전트가 수행해야 할 책임과 행동 범위를 정의한다.
명확한 역할 정의는 에이전트 간 책임 충돌을 줄이고,
전문화를 통해 작업 효율을 높인다.

## 2. Task Description (작업 설명)

instruction은 에이전트가 수행해야 할 구체적인 작업을 포함한다.
이때 작업은 가능한 한 명확하고 구체적으로 정의되어야 하며,
모호하거나 과도하게 큰 작업은 성능 저하로 이어질 수 있다.

## 3. Interaction Protocol (상호작용 규칙)
멀티 에이전트 환경에서는
에이전트 간의 통신 방식과 협업 절차를 명시하는 것이 중요하다.

* 어떤 에이전트에게 결과를 전달해야 하는지
* 어떤 조건에서 다른 에이전트를 호출해야 하는지
* 어떤 형식으로 정보를 전달해야 하는지

명확한 상호작용 규칙이 없을 경우,
시스템은 비효율적이거나 불안정한 행동을 보일 수 있다.
