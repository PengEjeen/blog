# Optimizer

활성화 함수와 옵티마이저 정리.

## 1. 활성화 함수 변천

### sigmoid → ReLU

sigmoid에서 기울기가 0에 수렴하는 부분은 학습 시간이 오래 걸린다 (기울기 ≈ 0이면 변화가 매우 느림).
반면 **ReLU**는 양수 영역에서 기울기가 1이라 학습이 빠름. → gradient descent 속도 향상.

> idea → code → experiment

### 죽은 ReLU 문제

일부 뉴런이 0 이외의 값을 출력하지 않는 문제. 해결 위해 변형들 등장:

- **LeakyReLU** — 음수 영역에 작은 기울기(α) 부여
- **RReLU (Randomized leaky ReLU)** — α를 학습 시 무작위 선택, 테스트 시에는 평균 사용
- **PReLU (Parametric leaky ReLU)** — α를 훈련하면서 학습. 대규모 이미지 데이터셋에서 ReLU보다 우수, 소규모에서는 과적합 위험.
- **ELU** — 음수 영역에서 지수적 감소

심층 신경망 은닉층 성능 순위 (실측):
**SELU > ELU > LeakyReLU > ReLU > tanh > logistic**

> 속도가 중요하면 ReLU.

## 2. 배치 정규화

학습 과정에서 각 배치 단위로 평균과 분산을 이용해 정규화.

### Gradient Clipping

그래디언트 폭주 문제 완화. 순환 신경망(RNN)은 배치 정규화 적용이 어려워 이 방법을 많이 사용.

## 3. 고속 옵티마이저

참고 - [twinw 블로그](https://twinw.tistory.com/247)

### 3.1 모멘텀 최적화

이전 그래디언트가 얼마였는지 기억. 매 반복에서 현재 그래디언트(학습률 곱한 후)를 모멘텀 벡터 m에 더하고 이 값을 빼는 방식으로 가중치 갱신.

- 새로운 hyperparam: `beta` (모멘텀 계수)
  - 0 = 마찰저항 큼, 1 = 마찰저항 없음
  - 기본값 **0.9**
- 식: `m ← β·m − lr·∇L`, `θ ← θ + m`

### 3.2 네스테로프 가속 경사 (NAG)

모멘텀 최적화의 변종. 현재 위치 θ가 아니라, 모멘텀 방향으로 조금 앞선 `θ + β·m`에서 그래디언트를 계산.

### 3.3 AdaGrad

가장 가파른 차원을 따라 그래디언트 벡터의 스케일을 감소. 즉 **적응적 학습률** — 가파른 차원일수록 더 빠르게 학습률 감소.

```
1. 그래디언트의 제곱을 벡터 s에 누적
2. 경사하강과 거의 같지만 스케일이 조정됨
```

문제: 신경망 훈련에서 너무 일찍 멈추는 경우가 있어 심층 신경망에는 잘 안 씀.

### 3.4 RMSProp

AdaGrad가 너무 빨리 느려져 전역 최적점에 수렴 못하는 위험을 보완. 가장 최근 반복의 그래디언트만 누적함으로써 문제 해결 (지수 감소 사용).

### 3.5 Adam

**모멘텀 최적화 + RMSProp**의 아이디어 결합.

- 그래디언트의 지난 지수 감소 평균 (모멘텀)
- 그래디언트 제곱의 지수 감소 평균 (RMSProp)
- hyperparam: `β1 = 0.9` (모멘텀 감쇠), `β2 = 0.999` (스케일 감쇠)

### 3.6 AdaMax

Adam이 s의 제곱근으로 파라미터 스케일을 낮추는데 (L2 norm 기반), AdaMax는 **L∞ norm**으로 바꾼 버전.
