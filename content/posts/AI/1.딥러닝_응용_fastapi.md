# 딥러닝 응용 FastAPI

딥러닝 모델을 FastAPI 위에 얹어 서빙하기 위한 정리.

## 0. 참고 튜토리얼

- [tech.osci.kr — FastAPI 파이썬으로 간단하게 웹 API 만들기](https://tech.osci.kr/fastapi-%ED%8C%8C%EC%9D%B4%EC%8D%AC%EC%9C%BC%EB%A1%9C-%EA%B0%84%EB%8B%A8%ED%95%98%EA%B2%8C-%EC%9B%B9-api-%EB%A7%8C%EB%93%A4%EA%B8%B0/)

## 1. 모델 로딩과 앱 수명주기 분리

요청마다 모델을 새로 로드하면 메모리/시간 낭비. `lifespan`(혹은 startup/shutdown 이벤트)에서 1회만 로드.

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.model = load_model("./model.pt")
    yield
    # 종료 시 정리
    del app.state.model

app = FastAPI(lifespan=lifespan)
```

## 2. 입력 스키마 검증 (Pydantic)

```python
from pydantic import BaseModel, Field

class PredictRequest(BaseModel):
    text: str = Field(..., min_length=1, max_length=2048)
    top_k: int = Field(5, ge=1, le=100)
```

## 3. 추론 엔드포인트와 예외 처리

```python
@app.post("/predict")
async def predict(req: PredictRequest):
    try:
        return {"result": app.state.model.infer(req.text, top_k=req.top_k)}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
```

## 4. 배포 설정

- CORS — `from fastapi.middleware.cors import CORSMiddleware`
- 멀티 워커 — uvicorn `--workers N` 또는 gunicorn + uvicorn worker
- GPU 모델 — 워커 수는 GPU 개수에 맞춰 1~2개로 제한 (메모리 충돌 방지)
