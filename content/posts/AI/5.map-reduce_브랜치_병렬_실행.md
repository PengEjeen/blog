# LangGraph map-reduce 브랜치 병렬 실행

공식 문서 - [Graph API: map-reduce and the Send API](https://langchain-ai.github.io/langgraph/how-tos/graph-api/#map-reduce-and-the-send-api)

## 왜 필요한가

작업 분할과 병렬 처리를 위해 필수.

1. **작업을 더 작은 하위 작업으로 분해** — 큰 입력을 chunk 단위로 쪼갬
2. **각 하위 작업을 병렬로 처리** — LangGraph의 `Send` API 로 동적 fan-out
3. **모든 하위 작업 결과 집계 (reduce)** — 다음 노드에서 합산/요약

## 핵심: `Send` API

기존의 정적 엣지(고정된 노드 간 연결)와 달리, `Send` 는 **런타임에 동적으로** 다음 노드의 입력을 만들 수 있다.

```python
from langgraph.types import Send

def map_node(state):
    # 입력 chunks 개수만큼 worker_node 인스턴스 fan-out
    return [Send("worker_node", {"chunk": c}) for c in state["chunks"]]
```

`worker_node` 가 chunk 개수만큼 동시 실행되고, 각각의 결과는 reducer에서 모인다 (state 의 `Annotated[..., operator.add]` 같은 reducer 정의 사용).

## 사용 시나리오

- 문서 N개를 각각 요약 후 최종 요약
- 여러 카테고리에 대해 동시에 LLM 분류
- 다중 source RAG 검색 → 결과 병합
