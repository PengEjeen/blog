# 1. Django 기본 동작

1. Django는 기본적으로 파일 이름 처리 로직이 있다.
2. `FileField`로 저장 시 자동 적용 (충돌 시 suffix가 붙거나 hash 처리).

# 2. 문제

사용자에게는 **원본 파일명**을 그대로 보여줘야 한다. 그러나 Django가 디스크에 저장할 때 파일명을 변경하므로 원본 이름이 손실된다.

# 3. 해결

파일 모델에 `original_name` 컬럼을 따로 둔다.

```python
class ConstructionContractFile(models.Model):
    contract = models.ForeignKey(
        "ConstructionContract",
        on_delete=models.CASCADE,
        related_name="files",
    )
    file = models.FileField(upload_to="contract_constructioncontract_file/")
    original_name = models.CharField(max_length=255)
```

# 4. 운영 패턴

- 업로드 시 `request.FILES["file"].name`을 `original_name`에 저장.
- 다운로드/표시 시 `original_name`을 노출.
- 디스크에는 Django가 자동 부여한 파일명으로 보존 → 동일 이름 충돌 방지.
