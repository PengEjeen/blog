# JSP (Java Server Pages)

> 학교 수업 / 사이드 프로젝트에서 다룬 JSP 학습 메모. (Notion 원본은 외부 객체 임베드라 본문 추출 불가 — 학습 항목으로 정리)

## 1. 기본 구조

- JSP 파일은 **서블릿으로 컴파일**되어 톰캣이 실행
- 스크립틀릿 `<% %>`, 표현식 `<%= %>`, 디렉티브 `<%@ %>`
- MVC 분리: JSP는 View 만 담당하고, Controller는 서블릿이 맡는 게 정석

## 2. 요청 / 응답 처리 흐름

1. 클라이언트 요청 → 서블릿 컨테이너
2. JSP → 서블릿 변환 (최초 1회)
3. `_jspService(request, response)` 실행
4. 결과 HTML 응답

## 3. 폼 데이터 처리

```jsp
<%
String username = request.getParameter("username");
String email = request.getParameter("email");
%>
```

- `request.getParameter()` 로 폼 값 수신
- 인코딩: `request.setCharacterEncoding("UTF-8")` 먼저 호출

## 4. 세션 / 쿠키 기반 상태 관리

- `session.setAttribute("user", userObj)` / `getAttribute`
- `Cookie` 객체 + `response.addCookie()`
- 보안: HttpOnly / Secure 플래그 / 만료 시간

## 5. JDBC 연동 기초

```jsp
<%
Class.forName("com.mysql.cj.jdbc.Driver");
Connection conn = DriverManager.getConnection(url, user, pwd);
PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE id=?");
ps.setInt(1, id);
ResultSet rs = ps.executeQuery();
%>
```

- 반드시 PreparedStatement (SQL 인젝션 방어)
- `try-with-resources` (Java 7+) 로 자원 누수 방지

## 실습 시 점검 포인트

- MVC 분리 (JSP에 비즈니스 로직 금지)
- 예외 처리 패턴 (`error-page` 디렉티브)
- 한글 깨짐 대응 (`pageEncoding="UTF-8"` + Tomcat URIEncoding)
