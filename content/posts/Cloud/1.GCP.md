# 1. Cloud Run 고정 IP 설정

참고 - [https://apeltop.github.io/Cloud-Run-Static-IP/](https://apeltop.github.io/Cloud-Run-Static-IP/)

기본 아이디어: Cloud Run은 egress 트래픽이 매번 다른 IP로 나가므로, **VPC Connector + Cloud NAT** 조합으로 정적 IP를 고정한다.

## 1.1 서브넷 생성

```bash
gcloud compute networks subnets create cloud-run-subnet \
  --range=10.124.0.0/28 \
  --network=default \
  --region=asia-northeast3
```

## 1.2 Serverless VPC Access Connector 생성

```bash
gcloud compute networks vpc-access connectors create cloud-run-connector \
  --region=asia-northeast3 \
  --subnet-project=pmis-468723 \
  --subnet=cloud-run-subnet
```

## 1.3 라우터 생성 + 정적 IP 예약 + NAT 설정

```bash
gcloud compute routers create cloud-run-router \
  --network=default \
  --region=asia-northeast3

gcloud compute addresses create cloud-run-origin-ip \
  --region=asia-northeast3

gcloud compute routers nats create cloud-run-nat \
  --router=cloud-run-router \
  --region=asia-northeast3 \
  --nat-custom-subnet-ip-ranges=cloud-run-subnet \
  --nat-external-ip-pool=cloud-run-origin-ip
```

## 1.4 Cloud Run 서비스 배포

```bash
gcloud run deploy pmis-cloud \
  --image=docker.io/namyeong/pmis-cloud:v1.1 \
  --vpc-connector=cloud-run-connector \
  --vpc-egress=all-traffic \
  --region=asia-northeast3 \
  --platform=managed \
  --allow-unauthenticated
```

`--vpc-egress=all-traffic` 가 핵심. 이걸 줘야 모든 외부 호출이 NAT를 거쳐서 고정 IP로 나간다.

## 1.5 Cloud NAT IP 확인

```bash
gcloud compute addresses describe cloud-run-origin-ip \
  --region=asia-northeast3 \
  --format="get(address)"
```

이 IP를 외부 시스템(파트너 API, 방화벽 화이트리스트 등)에 등록하면 된다.
