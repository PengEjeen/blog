# 2. 최소 기능을 갖춘 커널
## 2.1 부팅과정
1. 마더보드의 롬(ROM)에 저장된 펌웨어 코드를 실행. 시동 자체 시험 진행, 하드웨어 초기화작업
2. 부팅디스크 감지 후 운영체제 커널 부팅
x86 시스템에는 BIOS와 UEFI 펌웨어 표준이 존재
BIOS표준만 지원하는 운영체제만 만들거임

## 2.2 BIOS 부팅
부팅과정
1. 마더보드의 특별한 플래시 메모리로부터 BIOS 이미지 로드.
2. BIOS 이미지는 자가 점검 및 하드웨어 초기화 작업을 처리한 후, 부팅 가능한 디스크가 있는지 탐색.
3. 부팅 가능한 디스크가 있다면, 제어 흐름은 해당 디스크의 부트로더(디스크 맨 앞 512 바이트 크기의 실행 파일)에게 넘겨짐.
4. 부트로더는 커널 이미지가 디스크의 어느 주소에 저장되어있는지 알아낸 후 메모리에 커널 이미지 로드
5. CPU 전환 real mode(16비트) -> protected mode(32비트) -> long mode(64비트) 이후 레지스터, 메인메모리 사용 가능.
6. BIOS로부터 메모리 매핑 정보 등의 필요한 정보를 알아내어 운영체제 커널에 전달.

부트로더 작성 x, bootimage 툴 사용.

Multiboot 표준
Free software Foundation에서 multiboot라는 부트로더 표준을 개발.
리눅스에서는 GNU GRUB

## 2.5 컴파일 대상 정의
x86_64-blog_os.json파일 생성

```json
{
  "llvm-target": "x86_64-unknown-none",
  "data-layout": "e-m:e-i64:64-f80:128-n8:16:32:64-S128",
  "arch": "x86_64",
  "target-endian": "little",
  "target-pointer-width": "64",
  "target-c-int-width": "32",
  "os": "none",
  "executables": true,
  "linker-flavor": "ld.lld",
  "linker": "rust-lld",
  "panic-strategy": "abort",
  "disable-redzone": true,
  "features": "-mmx,-sse,+soft-float"
}
```

```bash
$ cargo build --target x86_64-blog_os.json
```

결과 `error[E0463]: can’t find crate for ‘core’`
rust 컴파일러가 core 라이브러리를 못찾음. core library 빌드 필요.
core library: Result와 Option 그리고 반복자 등 Rust의 기본적인 타입들을 포함.
모든 no_std 크레이트에 암시적으로 링크.

## 2.6 build-std기능
rust `core` 및 다른 표준 라이브러리 크레이트 버전 사용 대신, 이 기능을 이용.
완성된 기능이 아니기에 unstable한 기능으로 표기. nightly 버전의 rust 컴파일러에서만 이용가능.
