# 1.freestanding 실행파일
개요
운영체제 없이도 실행가능한 실행파일 = “freestanding 실행파일”, “bare-metal 실행파일”

단계
## 1.1 rust 표준 라이브러리 링크 해제하기
#![no_std] 속성 사용

## 1.2 패닉 (Panic) 시 호출되는 함수 구현하기
#[panic_handler] 속성 사용
Language item - 컴파일러가 내부적으로 요구하는 특별한 함수 및 타입들
eh_personality language item - 스택 되감기(stack unwinding)을 구현하는 함수.

## 1.3 스택 되감기 해제
Cargo.toml파일에 코드 추가
//cargo build 실행에서 패닉 시 실행 종료
[profile.dev]
panic = “abort”

//cargo build --release 실행에서 패닉 시 실행 종료
[profile.release]
panic = “abort”

> cargo build 실행 결과
error: requires ‘start’ lang_item

## 1.4 start 속성(start language item)
main함수 실행 이전에 먼저 호출되는 런타임 시스템
러스트의 경우 C 런타임 라이브러리 crt에서 실행
crt0가 작업 마친후start language item 실행
#![no_main]속성 사용
_start 함수를 실행 시작지점으로 대체
pub - 비공개규칙
#[no_mangle] //name mangling 해제
pub extern “C” fn_start() -> ! {
loop{}
}
실행 시 링커 오류발생

## 1.5 링커 오류
링커 - 컴파일러가 생성한 코드들을 묶어 실행파일로 만드는 프로그램.
링커가 C 런타임 시스템을 이용할 것이라고가정했지만 우리의 크레이트는 그렇지 않기 때문에 오류발생.
링커에게 C 런타임을 링크하지 말라고 알려줘야함.

1. 크레이트 컴파일 대상 기기를 bare metal 기기로 설정.
thumbv7em-none-eabihf target triple
//none - 운영체제가동작하지 않음.
rustup에서 해당 시스템 환경을 추가.
rustup target add thumbv7em-none-eabihf
cargo build --target thumbv7em-none-a-eabihf
//--target - 해당 bare metal 시스템을 목푤 크로스 컴파일 할 것을 cargo에게 알려줌
해당 target triple을 목표로 하는 freestanding 실행파일 생성 가능.

2. 링커에 특정 인자 전달.
#Linux
cargo rustc – -C link-arg=-nostartfiles
